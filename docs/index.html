The /notnow Protocol

Simple Commands, Powerful Results

NotNow works with ANY GitHub repository. No setup required, no database needed, no external services.

Just add comments to your issues using the /notnow protocol, and NotNow transforms them into a rich task management system.

All data stays in GitHub - your issues and comments ARE the database. This means zero vendor lock-in and complete data portability.

Example Commands:

/notnow init
Initialize an issue for NotNow tracking
/notnow subtask add "Write unit tests" --estimate 2h
Add a subtask with time estimate
/notnow start -d "Working on authentication"
Start tracking time with description
/notnow complete st1 --time 1h30m
Complete subtask and log time spent
/notnow tags add urgent,security
Add tags to categorize issues
/notnow priority high
Set issue priority level

Get Started in Minutes

1

Install NotNow

Clone the repository and build the application:

git clone https://github.com/MRIIOT/NotNow.git cd NotNow dotnet build NotNow.Quake/NotNow.Quake.csproj
2

Configure GitHub Access

Create a GitHub personal access token with repo scope and add it to appsettings.json:

{ "GitHub": { "PersonalAccessToken": "ghp_your_token", "Owner": "your-username", "Repository": "your-repo" } }
3

Launch & Use

Run NotNow and press Ctrl+~ to toggle the interface:

dotnet run --project NotNow.Quake

That's it! Start managing your GitHub issues with style.
